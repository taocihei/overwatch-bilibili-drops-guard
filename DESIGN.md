# 设计说明

## 产品定位

守望先锋 B 站直播挂宝是一个面向普通 Windows 用户的本地桌面工具。核心目标是让用户完成登录、确认直播间、后台计时、查看任务进度和领取奖励，不需要理解接口、线程或浏览器调试细节。

## 设计原则

- 操作路径要短：默认直播间、自动识别任务、自动保存配置。
- 语言要面向用户：用“后台计时”“当前进度”“还差几分钟”“已领取”这类直观表达。
- 反馈要清楚：任务进度和运行日志分开显示，优先展示用户最关心的奖励状态。
- 风险要克制：停止、清空、手动领取等操作保持明确但不过度打扰。
- 公开信息要干净：仓库只保留用户文档、源码和必要资源，不放内部调试记录。

## 页面结构

- 顶部：软件名称、版本号、免费声明和开源地址。
- 登录区：Cookie 获取入口、只打开登录页、账号状态提示。
- 配置区：直播间号、后台计时路数、刷新间隔、自动领取开关、通知 URL。
- 任务区：显示当前日期可挂奖励、每个奖励的进度和领取状态。
- 状态区：显示后台计时路数是否正常运行，必要时可展开查看每一路。
- 日志区：默认收起或压缩，只展示辅助排错信息。

## 视觉风格

- 整体接近柔和的 iOS 风格，避免工程样板感。
- 使用浅色背景、柔和圆角、低对比边框和清晰层级。
- 主要操作按钮使用稳定的绿色，停止类操作使用低饱和红色。
- 文字密度要适合工具软件，避免大面积装饰和无用说明。

## 用户提示

- 软件完全免费，任何购买行为都应向商家退款。
- 赞助只是对作者的一次点赞，不会带来任何功能、成功率或支持优先级提升。
- 默认直播间可直接使用，用户也可以手动填写 B 站直播间链接或房间号。
- 如果自动识别失败，应提供手动获取房间号和任务 ID 的说明。

## 验证要求

- 修改界面后运行单元测试。
- 涉及二维码或赞助图时，必须验证支付宝和微信二维码仍可识别。
- 涉及打包时，必须验证 exe 能打开，且不缺少 Tk/Tcl 运行目录。
